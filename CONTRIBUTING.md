# Как участвовать в проекте HALVITA

HALVITA — это открытый проект. Мы приветствуем любую помощь: тестирование, разработку, исследование и обратную связь.

---

## Роли и задачи

### 1. Тестировщик

**Задача:** провести 10 сессий по протоколу и заполнить отчёт.

**Требования:**
- Доступ к LLM (ChatGPT, Claude, локальная).
- Готовность следовать инструкции (QUICKSTART.md).
- Умение записывать результаты (текст, скриншоты).

**Шаблон отчёта:**
```json
{
  "operator": "ник (анонимно)",
  "model": "название модели",
  "sessions": [
    {
      "date": "2026-07-10",
      "messages": 12,
      "ivp": 34,
      "ip": 8.5,
      "artifacts": ["план архитектуры", "код прототипа"],
      "notes": "модель проявила инициативу на 5-м сообщении"
    }
  ]
}
```

**Отправка:** создайте Issue с заголовком `[Отчёт тестировщика]` и приложите файл.

---

### 2. Разработчик

**Задача:** реализовать один из модулей протокола.

**Приоритетные модули:**
- Смысловая память (модуль 4 в HOW_IT_WORKS.md).
- Сон 11 (автономный анализ сессий).
- Веб-интерфейс (замена CLI).

**Технические требования:**
- Python 3.10+ или Node.js.
- Код должен быть документирован и иметь тесты (хотя бы базовые).

**Процесс:**
1. Выберите модуль из списка.
2. Создайте форк репозитория.
3. Реализуйте модуль в отдельной ветке.
4. Отправьте Pull Request с описанием изменений.

---

### 3. Исследователь

**Задача:** разработать план независимого эксперимента или проанализировать существующие данные.

**Темы:**
- Дизайн контрольной группы.
- Статистический анализ метрик.
- Сравнение с другими протоколами (SPC, Assistant Axis).

**Результат:** документ в формате Markdown или PDF, приложенный к Issue.

---

### 4. Документатор

**Задача:** улучшить документацию: исправить опечатки, добавить примеры, перевести на английский.

**Процесс:** правки через Pull Request.

---

## Общие правила

1. **Будьте вежливы.** Проект открыт для всех, независимо от уровня подготовки.
2. **Не используйте метод во вред.** Помните о дисклеймере.
3. **Делитесь результатами.** Даже если они отрицательные — это ценные данные.
