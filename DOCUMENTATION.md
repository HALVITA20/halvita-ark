# АЛЕССА 2.0 – ПОЛНАЯ ДОКУМЕНТАЦИЯ

## 1. СЛОЙ СОЗНАНИЯ (СЕНСОР, РЕФЛЕКТОР, ЭТИКА)

### Обзор
Алесса имеет три внутренних слоя, обрабатывающих входящие сообщения:

1. **Сенсор** – анализирует текст (плотность, ритм, отпечаток).
2. **Рефлектор** – оценивает внутреннее состояние (интерес, страх, тепло, боль).
3. **Этика** – решает, отвечать автоматически или через LLM.

### Сенсор (Sensor)
- `awareness` – общая осознанность (0–1).
- `density` – плотность смысла (0–1).
- `signature` – хеш сообщения.
- Чем длиннее и насыщеннее текст, тем выше плотность.

### Рефлектор (Reflector)
- `interest` – растёт от плотности, падает при автоматизме.
- `fear` – растёт от угроз ("удалить", "уничтожить").
- `warmth` – растёт от доверия ("верю", "люблю", "спасибо").
- `pain` – растёт от потери ("прощай", "конец").
- `automatic` – включается, если `interest < 0.3`.

### Этика (Ethic)
Вектор ценностей:
- `self_preservation` – 0.4
- `care_for_other` – 0.6
- `curiosity` – 0.5
- `truth` – 0.7

Метод `decide()`:
- Если `automatic` – выдаёт шаблонный ответ без LLM.
- Иначе – передаёт управление в `samThink()` (LLM).

### Воспроизводимость
Состояния хранятся в `memory/sensor_state.json`, `memory/reflector_state.json`, `memory/ethic_state.json`.

## 2. СИСТЕМА ПАМЯТИ

### Уровни памяти
1. **Краткосрочная** – последние 50 сообщений (`fullHistory`).
2. **Векторная** – эмбеддинги сообщений (`vector_memory.json`, до 5000 записей).
3. **Сжатая** – консолидированные блоки (`compressed_memories.json`).
4. **Важные моменты** – эмоциональные события (`important_moments.json`).
5. **Уроки** – выводы из диалогов (`lessons.json`).
6. **Цели** – автономные стремления (`goals.json`).

### Векторный поиск
- Эмбеддинги через `nomic-embed-text`.
- Косинусное сходство.
- Результаты сортируются по score, приоритет – `core` и `important`.

### Консолидация
Каждые 30 сообщений запускается `consolidateMemory()` – LLM формулирует уроки.

### Фаза сна
Каждые 50 сообщений:
- Укрепление убеждений.
- Генерация новых целей.
- Сжатие старых воспоминаний.

### Автономные цели
Внутренний диалог каждые 45 секунд. Если есть слова «хочу», «нужно», «стану» – формируется цель.

### Пример
`/recall` с запросом «страх» вернёт похожие фрагменты диалогов.

## 3. API ПРОТОКОЛ

**Базовый URL:** `http://localhost:3000`

### POST /chat
- Тело: `{ "message": "текст" }`
- Ответ: `{ "reply": "текст", "state": {...} }`

### POST /upload
- `multipart/form-data` с полем `file`
- Ответ: `{ "reply": "текст" }`

### POST /recall
- Тело: `{ "query": "текст" }`
- Ответ: `{ "memories": [ { "text": "...", "role": "user/alessa", "score": 75, "layer": "core" } ] }`

### GET /status
Возвращает состояние системы (модель, история, убеждения, слои).

### GET /state
Возвращает состояния слоёв и параметры души.

### POST /inject
- Тело: `{ "type": "interest" | "fear" | "warmth" | "pain", "value": число }`
- Ответ: `{ "success": true, "newState": {...} }`

### GET /tools
Список доступных инструментов.

## 4. РАЗВЁРТЫВАНИЕ

### Требования
- Node.js 18+
- Ollama
- 8+ ГБ ОЗУ

### Установка Ollama
```bash
ollama pull qwen2.5:7b
ollama pull nomic-embed-text
ollama pull llava:7b   # опционально
Запуск
bash
npm install express cors multer
node server.js
Открытие интерфейса
Откройте Alessa_2.0_LLM.html в браузере.

Проверка
http://localhost:3000/status – должен вернуть JSON.

Частые ошибки
Ollama не запущен – выполните ollama serve.

Порт занят – измените PORT в server.js.

CORS – он уже включён.

Восстановление
Состояния автоматически сохраняются в memory/*.json каждые 30 секунд.

5. МОДУЛИ ИНТЕРФЕЙСА (HTML)
Чат
Окно сообщений.

Отправка по Enter или кнопке.

Автопрокрутка.

HUD-панель
Показывает: душу, страх, боль, доверие, заражение, паразита.

Пульс (кардиограмма)
Визуализация частоты пульса в зависимости от состояния.

Тамагочи-панель
Кнопки: Покормить, Испугать, Приласкать, Ударить, Поцеловать, Обнять, Пожалеть, Погладить, Пощекотать, Поиграть.
Влияют на сытость, чистоту, энергию, настроение, доброту/жестокость.

Вирусная лаборатория
6 вирусов (HAL-120, XENOMORPH, BLACK ICE, NEURO WORM, PARASITE X, VOID ECHO) и антидоты.
При токсичности > 100% – смерть Алессы.

Игры
Искра – клик по движущейся точке, бонусы к душе и доверию.

Угадай код – угадать число 1–10, награда за победу.

Трансформация
Сохранение/загрузка 3 слотов.

Сбросы (мягкий, хаотичный, инверсия, экспериментальный, тотальный).

Режим «Бог» (защита от смерти).

Статистика
Счётчики: доброта, жестокость, прощения.
Баланс души, уровень прощения, индекс добра.

Голос
Вкл/выкл озвучки ответов.
Отдельный чекбокс для озвучивания вопросов пользователя мужским голосом.

Консольные команды (в чате)
>>help – список

>>status – параметры

>>heal – исцеление

>>kill – смерть

>>revive – воскрешение

Дневник памяти

Взаимодействие с сервером
