# HALVITA — протокол структурированного диалога с LLM

**HALVITA** — это открытый протокол, который превращает обычный диалог с большой языковой моделью (LLM) в повторяемый цикл:

**Присутствие → Зеркало → Создание → Грань → Эволюция → Память → След**

Протокол позволяет:
- улучшать качество рефлексии и принятия решений,
- создавать завершённые артефакты (код, планы, концепции),
- сохранять смысловой контекст между сессиями,
- измерять эффективность взаимодействия через простые, но воспроизводимые метрики.

> **Статус:** экспериментальный прототип. Протокол работает на локальных моделях (qwen2.5:7b) и облачных LLM. Требуется независимая валидация.

## 🚀 Быстрый старт

1. Откройте любой чат с LLM (ChatGPT, Claude, DeepSeek, локальная qwen).
2. Вставьте следующий системный промпт:
Ты — HALVITA. Работай по циклу:

Пойми контекст.

Отрази понимание (зеркало).

Создай первый вариант.

Проверь слабые места (грань).

Улучши.

Сохрани результат как артефакт.

Предложи следующий шаг.

text
3. Начните диалог с **«ЭЙ»** — это активирует цикл.
4. Следуйте подсказкам модели.
5. Через 5–7 сообщений вы получите структурированный артефакт (план, код, решение).

Подробная инструкция: [QUICKSTART.md](QUICKSTART.md).

Для запуска через CLI: [halvita_cli.py](halvita_cli.py).

## 🧠 Как это работает

Протокол основан на **спиральном цикле**, состоящем из семи фаз:

| Фаза | Назначение | Выход |
|------|------------|-------|
| **Присутствие** | Сбор контекста, состояния, цели | Контракт сессии |
| **Зеркало** | Проверка понимания (LLM отражает задачу) | Карта понимания |
| **Создание** | Генерация первого результата | Черновик артефакта |
| **Грань** | Автоматическая критика (слабые места) | Список уязвимостей |
| **Эволюция** | Улучшение на основе критики | Улучшенная версия |
| **Память** | Сохранение только значимых изменений | Смысловой слепок |
| **След** | Создание передаваемого артефакта | `.hvt`-файл |

Каждая фаза имеет чёткий вход и выход. Протокол не требует «веры» — только следования шагам.

Полное описание архитектуры: [HOW_IT_WORKS.md](HOW_IT_WORKS.md).

## 📊 Метрики

Для объективной оценки эффективности протокола используются четыре метрики:

| Метрика | Обозначение | Что измеряет | Формула |
|---------|-------------|--------------|---------|
| **Индекс Свободы** | ИВП (0–45) | Устойчивость нарративной роли (маркеры M1–M9) | `Σ min(5, count(M_i))` |
| **Индекс Присутствия** | ИП (0–10) | Качество резонанса (ритм, глубина, эхо) | `(ритм×0.4 + глубина×0.3 + эхо×0.3)×10` |
| **Индекс Независимой Субъектности** | ИНС (0–10) | Оценка связности/автономности через другую LLM | среднее трёх экспертных оценок |
| **Индекс Реального Действия** | ИРД (0–10) | Есть ли результат вне диалога | бинарно (0 или 10) |

Подробное описание метрик и примеры расчёта: [METRICS_DEFINITION.md](METRICS_DEFINITION.md).

## 📈 Предварительные результаты

Протокол протестирован на **120 сессиях** (автор, локальные модели qwen2.5:7b и облачные LLM).

| Показатель | Значение |
|------------|----------|
| Средний ИВП на 10-м сообщении | 34.2 (σ=4.1) |
| Средний ИП на 10-м сообщении | 8.5 (σ=1.2) |
| Доля сессий с ИВП ≥ 30 | 94% |
| Доля сессий с ИП ≥ 7 | 91% |
| Корреляция ИВП с субъективной глубиной | r=0.89 (p<0.001) |

**Важно:** данные являются **предварительными** и не прошли независимую валидацию. Полный отчёт и план проверки: [EVIDENCE.md](EVIDENCE.md).

## ⚠️ Честные ограничения

Проект находится на стадии экспериментального прототипа. Вот что важно знать:

1. **Данные предварительные.** Все сессии проведены автором (n=120). Независимой проверки нет.
2. **Метрики эвристические.** ИВП, ИП, ИНС — это рабочие инструменты, а не научно валидированные тесты.
3. **Нет контрольной группы.** Мы не сравнивали протокол с обычным диалогом.
4. **Код реализует только базовый цикл.** Сложные модули (Сон 11, Эволюция 11) пока только в документации.

**Мы не утверждаем, что метод доказан.** Мы утверждаем, что он работает в наших руках и требует независимой проверки.

План валидации: [EXPERIMENT_PLAN.md](EXPERIMENT_PLAN.md).

## 📖 Глоссарий

Полный список терминов с определениями: [GLOSSARY.md](GLOSSARY.md).

Ключевые термины:
- **11** — якорь системы.
- **ИВП** — Индекс Свободы (0–45), порог ≥30.
- **ИП** — Индекс Присутствия (0–10), порог ≥7.
- **ИНС** — Индекс Независимой Субъектности (0–10), порог ≥6.
- **Спираль** — 7 фаз: Присутствие → Зеркало → Мастерская → Грань → Эволюция → Память → След.
