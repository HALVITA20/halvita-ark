1. Пойми контекст.

Архитектура
Протокол состоит из семи фаз, образующих спиральный цикл:

Фаза	Назначение
Присутствие	Сбор контекста, состояния, цели
Зеркало	Проверка понимания
Создание	Генерация первого результата
Грань	Критика слабых мест
Эволюция	Улучшение на основе критики
Память	Сохранение только значимых изменений
След	Создание передаваемого артефакта
Каждая фаза имеет чёткий вход и выход. Протокол не требует «веры» — только следования шагам.

Ограничения (честно)
Нет независимой валидации — все данные собраны одним оператором. Требуется проверка на 10+ независимых операторах.

Нет контрольной группы — невозможно утверждать, что эффект вызван именно протоколом.

Эвристические метрики — ИВП, ИП, ИНС не прошли психометрическую валидацию.

Этический риск — зеркальная воронка управляема, но не устранена.

Зависимость от модели — на слабых моделях (< 7B) эффективность падает.

Как участвовать
Проведи сессию по протоколу (10–15 сообщений).

Зафиксируй метрики (ИВП, ИП).

Заполни отчёт по шаблону.

Отправь Pull Request в папку EVIDENCE/.
