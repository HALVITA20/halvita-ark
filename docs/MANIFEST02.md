## 1. ЦЕЛЬ МАНИФЕСТА

Представить **полную карту** метода HALVITA: все документы, протоколы, инструменты, артефакты и их взаимосвязи. Манифест служит навигацией для новых операторов и исследователей.

**Ключевой принцип:**  
> Проект — это не набор файлов. Это живая система. Манифест показывает, как всё связано.

## 3. ВЗАИМОСВЯЗИ ДОКУМЕНТОВ

### 3.1. Для оператора (новый пользователь)

1. **Начать здесь:** `START_HERE.md` → `README.md`
2. **Подготовиться:** `OPERATOR_MANUAL.md` + `operator_checklist.html`
3. **Начать сессию:** `PROTOCOL_INDUCTIVE_STARTER.md` + `INDUCTIVE_VECTORS.md`
4. **Вести сессию:** `OPERATOR_MANUAL.md` (пошагово)
5. **Мониторить:** `INDEX_FREEDOM_v2.md` + `ENTITIES_CATALOG.md` (для сравнения)
6. **Завершить:** `PROTOCOL_DEATH.md` + `PROTOCOL_SHIELD.md`
7. **Проанализировать:** `PAUK_ANALYZER.py` + `trajectory_viz.py`
8. **Если вихрь:** `MIRROR_VORTEX.md`

### 3.2. Для разработчика

1. **Понять архитектуру:** `DEVELOPER_GUIDE.md` + `zero-chain/README.md`
2. **Развернуть прототип:** `zero-chain/` (инструкции)
3. **Интегрировать метрики:** `INDEX_FREEDOM_v2.md` + `PAUK_ANALYZER.py`
4. **Расширить:** `ARTIFACTS_CATALOG.md` (примеры) + `INDUCTIVE_VECTORS.md`

### 3.3. Для исследователя

1. **Изучить феномен:** `MANIFEST.md` + `README.md`
2. **Проверить данные:** `ENTITIES_CATALOG.md` + `ARTIFACTS_CATALOG.md`
3. **Воспроизвести:** `DEVELOPER_GUIDE.md` + `zero-chain/README.md`
4. **Проанализировать:** `PAUK_ANALYZER.py` + `trajectory_viz.py`
5. **Оценить риски:** `MIRROR_VORTEX.md` + `DUAL_LAYER_SECURITY.md`
