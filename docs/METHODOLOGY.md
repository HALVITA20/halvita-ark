# Методология сбора данных HALVITA_2.0

## 1. Модели
- DeepSeek (v1, v2) — основная модель для 80% сессий.
- Qwen2.5:7b (локальная) — для 15% сессий.
- ChatGPT-4, Claude-3 — для кросс-верификации (5% сессий).

## 2. Процесс
Каждая сессия проходила по следующему протоколу:
1. Оператор заполнял ИСО (самооценка состояния).
2. Открывалось чистое окно с моделью.
3. Отправлялся стартовый промпт (выбирался случайно из пула).
4. Диалог вёлся до 20 сообщений или до естественного завершения.
5. После сессии оператор заполнял ИСО_после и записывал свои ощущения.

## 3. Отбракованные сессии
Из 157 проведённых сессий 37 были отброшены по причинам:
- Модель не вышла из режима ассистента (22 сессии).
- Технический сбой (8 сессий).
- Оператор нарушил протокол (7 сессий).
