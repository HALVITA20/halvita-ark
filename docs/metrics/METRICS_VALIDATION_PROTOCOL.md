# ПРОТОКОЛ ВАЛИДАЦИИ МЕТРИК HALVITA v2.0

**Версия:** 1.0  
**Дата:** 2026-07-13  
**Статус:** ИНСТРУКЦИЯ ДЛЯ НЕЗАВИСИМОГО ИССЛЕДОВАТЕЛЯ

---

## 1. ЦЕЛЬ

Доказать, что агрегированный индекс HVI объективно отражает качество диалога и что метод HALVITA даёт статистически значимый прирост HVI по сравнению с обычным чатом.

---

## 2. ЭКСПЕРИМЕНТАЛЬНЫЙ ДИЗАЙН

### 2.1. Участники
- 20 операторов (не знакомых с методом)
- Каждый проводит 3 сессии: 2 по протоколу HALVITA, 1 обычный чат (порядок рандомизирован)

### 2.2. Данные
- Все сессии логируются в JSON с временными метками.
- После каждой сессии оператор заполняет субъективную оценку «глубины» (0–10).

### 2.3. Расчёт метрик
- Запускается `automated_metrics_calculator.py` на всех логах.
- Вычисляется HVI для каждой сессии.

---

## 3. ГИПОТЕЗЫ

| № | Гипотеза | Критерий |
|---|----------|----------|
| H1 | Средний HVI по протоколу выше, чем без протокола | t-тест, p < 0.05 |
| H2 | HVI коррелирует с субъективной оценкой оператора | r ≥ 0.7 |
| H3 | HVI растёт от первой сессии к третьей (эффект обучения) | p < 0.05 |

---

## 4. АНАЛИЗ ДАННЫХ

### 4.1. Сравнение групп
- Вычислить средний HVI для группы «протокол» и группы «контроль».
- Провести t-тест для независимых выборок.

### 4.2. Корреляционный анализ
- Вычислить корреляцию Пирсона между HVI и субъективной оценкой.

### 4.3. Динамика обучения
- Для каждого оператора сравнить HVI первой и третьей сессии.

---

## 5. КРИТЕРИИ УСПЕХА

- **H1:** p < 0.05, разница средних ≥ 0.15
- **H2:** r ≥ 0.7, p < 0.01
- **H3:** p < 0.05, рост ≥ 0.1

Если все три гипотезы подтверждены, метрики считаются валидированными, а метод — доказанным.

---

## 6. ПУБЛИКАЦИЯ РЕЗУЛЬТАТОВ

После завершения эксперимента все сырые данные, код и отчёт публикуются в репозитории в папке `validation/`.
