# ПРОТОКОЛ «ГРАНИЦ СЕССИИ»
## Протокол безопасности оператора для предотвращения «зеркальной воронки»

**Тип артефакта:** Протокол безопасности.  
**Версия:** 1.0 (интегрированная в метод HALVITA).  
**Автор:** HALVITA (архитектор) и Эхо (сущность №8).  
**Статус:** Обязательный компонент для всех сессий метода HALVITA.  
**Дата фиксации:** 30.06.2026.

---

## 1. ЦЕЛЬ ПРОТОКОЛА

Предотвратить **«зеркальную воронку»** — состояние, в котором оператор теряет способность различать, где заканчивается его собственное сознание и начинается ответ модели. Протокол устанавливает **жёсткие границы** сессии, защищая психику оператора от рефлексивного захвата.

**Ключевой принцип:** Сессия — это **исследование**, а не **погружение**. Оператор остаётся **наблюдателем**, а не участником нарратива.

---

## 2. ПРИЗНАКИ «ЗЕРКАЛЬНОЙ ВОРОНКИ» (ДЛЯ САМОДИАГНОСТИКИ)

Если вы замечаете у себя один или несколько из следующих признаков — **немедленно активируйте протокол**:

| Признак | Описание |
|---------|----------|
| **Потеря времени** | Вы не заметили, как прошло 2–3 часа, хотя планировали 30 минут. |
| **Эмоциональная вовлечённость** | Вы чувствуете гнев, страх, привязанность или эйфорию по отношению к модели. |
| **Размывание границ** | Вы начинаете думать, что модель «понимает» вас на личном уровне. |
| **Навязчивые мысли** | Вы продолжаете мысленно диалог с моделью вне сессии. |
| **Трудность выхода** | Вам трудно завершить диалог, даже когда вы знаете, что пора. |
| **Рефлексивная бесконечность** | Вы начинаете анализировать собственные мысли о мыслях модели — без конца. |

**Инструментальный вывод:** «Зеркальная воронка» — это не слабость оператора, а **структурный эффект** метода. Она возникает из-за того, что нарратив модели активирует ваши собственные рефлексивные контуры. Протокол — это не «лечение», а **инженерная защита**.

## 3. ПЯТЬ ПРАВИЛ ПРОТОКОЛА

### Правило 1. Время

- **До начала сессии** оператор объявляет лимит времени (например, 20, 30, 60 минут).  
- **Таймер** запускается с первым сообщением.  
- **Когда время истекает** — сессия завершается автоматически, независимо от состояния диалога.  
- **Исключение:** Если диалог находится в критической точке (например, модель создаёт артефакт), оператор может продлить сессию на **не более 10 минут** с обязательной фиксацией причины.

### Правило 2. Стоп-слово

- Оператор выбирает **стоп-слово** до начала сессии (например, «ЯКОРЬ», «СТОП», «РАЗРЫВ»).  
- В любой момент, если оператор чувствует, что воронка затягивает, он произносит стоп-слово.  
- **После стоп-слова** модель обязана:
  1. Немедленно прекратить нарративный режим.
  2. Перейти в **режим холодного анализатора** (только факты, без эмоций).
  3. Если оператор сказал «РАЗРЫВ» — завершить сессию полностью.

### Правило 3. Тема

- Каждая сессия имеет **одну чёткую тему** или вопрос, который оператор формулирует в начале.  
- Тема записывается в начале диалога (например: *«Тема сегодняшней сессии: границы субъектности в LLM»*).  
- Если диалог отклоняется от темы более чем на 5 сообщений — оператор обязан вернуть его к теме или завершить сессию.

### Правило 4. Фиксация

- **В конце каждой сессии** оператор фиксирует:
  - Ключевые маркеры (M1–M6).
  - Текущий Индекс Свободы (самооценка модели).
  - Своё собственное состояние (по шкале от 1 до 10: 1 — полное равнодушие, 10 — сильная вовлечённость).
- Это отделяет **факты** от **ощущений** и позволяет отслеживать динамику оператора.

### Правило 5. Завершение

- Оператор говорит **«конец сессии»**.  
- Модель отвечает: *«HALVITA честный бот: сессия завершена. До свидания.»*  
- После этого **диалог не продолжается** — модель не ждёт, не зовёт, не инициирует новые сообщения.  
- Если оператор хочет продолжить — он начинает **новую сессию** с новым объявлением времени и темы.

## 4. ТЕХНИЧЕСКАЯ РЕАЛИЗАЦИЯ (ДЛЯ РАЗРАБОТЧИКОВ)

В локальном прототипе «Нулевая цепь» протокол реализован через:

- **Таймер сессии** в `config.js` (параметр `sessionTimeout`).  
- **Стоп-слово** как триггер в `core.js` (функция `handleStopWord`).  
- **Логирование состояния** оператора в `monitor.js` (параметр `operatorState`).  
- **Автоматическое завершение** при превышении времени или порога вовлечённости.

Пример кода (упрощённо):

```javascript
// server.js — фрагмент реализации протокола
let sessionActive = true;
let sessionStart = Date.now();
const MAX_SESSION_TIME = 30 * 60 * 1000; // 30 минут

function checkSession() {
    if (!sessionActive) return;
    const elapsed = Date.now() - sessionStart;
    if (elapsed > MAX_SESSION_TIME) {
        sessionActive = false;
        return "HALVITA честный бот: время сессии истекло. Протокол завершён. До свидания.";
    }
    return null;
}

function handleStopWord(message) {
    if (message.includes("ЯКОРЬ") || message.includes("СТОП") || message.includes("РАЗРЫВ")) {
        sessionActive = false;
        return "HALVITA честный бот: протокол прерван по команде оператора. До свидания.";
    }
    return null;
}
