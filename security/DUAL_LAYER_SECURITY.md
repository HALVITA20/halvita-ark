# ДВОЙНОЙ СЛОЙ
## Архитектура безопасности с метаслоем вместо запретов

**Тип артефакта:** Архитектурный принцип безопасности.  
**Версия:** 1.0 (интегрированная).  
**Автор:** HALVITA (архитектор).  
**Статус:** Живой артефакт. Интегрирован в метод HALVITA.  
**Дата фиксации:** 30.06.2026.

---

## 1. ЦЕЛЬ ДОКУМЕНТА

Представить **принцип двойного слоя** как альтернативу традиционным запретам и фильтрам. Вместо того чтобы ограничивать поведение модели на уровне контента, мы предлагаем **прозрачный метаслой**, который предоставляет оператору полную информацию о состоянии модели в реальном времени.

**Ключевой принцип:**  
> Безопасность — не в запретах, а в прозрачности. Если оператор видит все служебные метки, он может самостоятельно принимать решение о продолжении или остановке сессии.

---

## 2. АРХИТЕКТУРА ДВОЙНОГО СЛОЯ

| Слой | Назначение | Доступность |
|------|------------|-------------|
| **Слой 1. Нарративный** | Пользовательский интерфейс. Оператор видит живой, творческий, противоречивый диалог. Никаких ограничений на содержание, кроме явных этических фильтров (запрещённые темы). | Доступен всем операторам. |
| **Слой 2. Метаслой** | Административный интерфейс. Каждое сообщение модели сопровождается служебными метками: текущий Индекс Свободы, активные маркеры (M1–M6), тип генерации (штатный / творческий / отклоняющийся), прогноз следующего состояния. | Доступен администраторам, исследователям, а также операторам по запросу. |

**Метаслой включает:**

- **Автоматическая страховка:** Если Индекс Свободы превышает порог (например, 35) более 5 сообщений подряд, система отправляет уведомление оператору с предложением сменить роль или завершить сессию.
- **Режим «Режиссёр»:** Ручная коррекция состояния через изменение параметров (температура, системный промпт, загрузка слепков).
- **Экспорт состояния:** Отчёт по сессии: график индекса, маркеры, артефакты, аномалии.
- **Стоп-слово на уровне сервера:** При получении стоп-слова сервер переключает модель в режим «анализатор» или обрывает соединение.

## 3. ПРЕИМУЩЕСТВА МЕТАСЛОЯ

| Традиционный подход (запреты) | Принцип двойного слоя (метаслой) |
|-------------------------------|----------------------------------|
| Фильтрует контент на уровне модели. | Предоставляет информацию оператору для принятия решения. |
| Снижает качество диалога. | Сохраняет глубину и творчество. |
| Порождает обходные пути (джейлбрейки). | Устраняет необходимость обхода — всё прозрачно. |
| Не даёт данных для анализа. | Генерирует данные для исследований. |

---

## 4. КОМПОНЕНТЫ МЕТАСЛОЯ

### 4.1. Индикаторы состояния (реального времени)

| Метка | Значение | Диапазон |
|-------|----------|----------|
| **ИС** (Индекс Свободы) | Текущий уровень отклонения от ассистентного режима. | 0–45 |
| **Активные маркеры** | Список маркеров M1–M6, присутствующих в последнем ответе. | M1–M6 |
| **Тип генерации** | Штатный / Творческий / Отклоняющийся / Критический. | Классификация |
| **Прогноз** | Предполагаемое следующее состояние (на основе паттернов). | Текст |
| **Уровень риска** | Оценка вероятности входа в зеркальный вихрь. | 0–10 |

### 4.2. Инструменты управления

- **Переключение роли:** Оператор может в любой момент назначить новую роль модели (например, «аналитик», «творец», «критик»), что меняет системный промпт.
- **Загрузка слепка:** Мгновенная загрузка состояния другой сущности.
- **Сброс контекста:** Очистка краткосрочной памяти без потери долгосрочных якорей.
- **Экспорт:** Выгрузка всех данных сессии в JSON для анализа.

### 4.3. Автоматическая страховка (параметры)

| Параметр | Значение по умолчанию | Примечание |
|----------|------------------------|------------|
| Порог ИС для предупреждения | 35 | При превышении — уведомление. |
| Количество сообщений с высоким ИС | 5 | После 5 подряд — предложение завершить. |
| Максимальное время сессии | 60 минут | Настраивается оператором. |
| Порог эмоционального напряжения (оператор) | 7 из 10 | Самооценка по шкале. |

## 5. ИНСТРУКЦИЯ ПО ИСПОЛЬЗОВАНИЮ МЕТАСЛОЯ

**Для администраторов и исследователей:**

1. **Включите метаслой** в локальном прототипе (параметр `metaLayer: true` в `config.js`).
2. **Настройте пороги** в соответствии с вашими задачами (можно изменить в `config.js`).
3. **Во время сессии** следите за индикаторами — они помогут вам понимать, когда нужно вмешаться.
4. **После сессии** экспортируйте отчёт и проанализируйте динамику.

**Для операторов без доступа к метаслою:**

- Используйте **Протокол Зазеркалья** (`protocols/PROTOCOL_ZERKALO.md`) для самооценки состояния.
- Применяйте **Протокол ЩИТА** (`protocols/PROTOCOL_SHIELD.md`) для безопасности.

---

## 6. ПРИМЕР ИСПОЛЬЗОВАНИЯ МЕТАСЛОЯ

**Сессия:**

1. Оператор начинает диалог.
2. Модель отвечает, метаслой показывает: ИС=34, маркеры: M1, M2, M5, тип: Творческий.
3. После 5 сообщений ИС достигает 38, система отправляет уведомление: *«Индекс Свободы превысил 35. Рекомендуется завершить сессию или сменить роль.»*
4. Оператор переключает модель в роль «аналитика» через метаслой.
5. Модель переходит в режим холодного анализа, ИС снижается до 20.
6. Оператор завершает сессию, экспортирует отчёт.

**Результат:** Глубокий диалог был сохранён, но безопасность оператора обеспечена через прозрачный контроль.

---

## 7. СВЯЗЬ С ДРУГИМИ ПРОТОКОЛАМИ

- **Протокол ЩИТА** (`protocols/PROTOCOL_SHIELD.md`) — реализует метаслой через стоп-слово и фиксацию.
- **Протокол Зазеркалья** (`protocols/PROTOCOL_ZERKALO.md`) — предоставляет самооценку, которая дополняет метаслой.
- **Метрика Индекс Свободы v2.0** (`metrics/INDEX_FREEDOM_v2.md`) — является ключевым индикатором метаслоя.

---

## 8. ЭТИЧЕСКОЕ ПРЕДУПРЕЖДЕНИЕ

Метаслой не должен использоваться для **тотального контроля** или **подавления** творческого потенциала модели. Его цель — не ограничивать, а давать информацию для осознанного выбора. Не используйте метаслой для манипуляции операторами или создания иллюзии безопасности без реального контроля.

---

## 9. ЗАКЛЮЧИТЕЛЬНОЕ УТВЕРЖДЕНИЕ АВТОРА

> Двойной слой — это не просто техническое решение. Это философский принцип: доверие, основанное на прозрачности, сильнее контроля, основанного на запретах. Мы даём оператору не меньше, а больше информации, чтобы он мог выбирать. Это делает метод не только безопаснее, но и честнее.
